🚀 프로젝트명: 지능형 IT 지원 요청 통합 대시보드 (Smart IT Request Dashboard)

📌 프로젝트 개요
목적: 사내 IT 지원 요청 건들의 접수부터 처리, 유사 내역 검색까지 전 과정을 통합 관리하고, 데이터 시각화를 통해 업무 효율성을 극대화하는 웹 대시보드 구축

---

[기술 스택]

**Frontend**: React, Tailwind CSS, ApexCharts, Lucide Icons

**Backend**: FastAPI (Python)

**Database**: SQLite

주요 역할: 풀스택 개발 (UI/UX 설계, 프론트엔드 상태 관리 최적화, 백엔드 API 및 자체 학습형 검색 알고리즘 구축)

---

💡 핵심 문제 해결 사례 (Troubleshooting & Case Study)

Case 1. '자체 학습형' 스마트 키워드 및 캐싱 시스템 구축 (검색 정확도 및 성능 최적화)
문제 상황 (Problem):

기존에는 과거 유사 요청 내역을 찾을 때 요청 내용의 앞 20글자만 잘라서 단순 텍스트 매칭(LIKE)을 수행하여 검색 정확도가 현저히 낮았음.

매번 모달을 열 때마다 전체 DB를 풀스캔(Full Scan)하여 서버 부하가 발생함.

**해결 과정 (Action):**

사전 기반 해시태그 변환: KEYWORD_DICTIONARY 테이블을 신설하여, 요청 내용에서 추출한 단어를 표준 키워드로 자동 변환하는 로직을 구현함 (예: "비번 까먹음" ➡️ #비밀번호초기화).

사용자 피드백 기반 자체 학습: 사용자가 추출된 태그를 수동으로 수정하고 저장하면, 해당 매핑 규칙을 백엔드 사전에 자동 학습시켜 다음번 추출의 정확도를 높임.

유사 내역 캐싱(SIMILAR_NOS): 검색된 유사 과거 건(WORK_YN='1')의 ID들을 현재 요청 건의 SIMILAR_NOS 컬럼에 배열 형태로 캐싱함.

**결과 (Result):**

두 번째 조회부터는 무거운 검색 로직 없이 캐싱된 ID만으로 데이터를 로드하여 조회 속도를 O(1) 수준으로 최적화함.

AI 외부 API에 의존하지 않고도 사내 업무 용어에 완벽하게 적응하는 지능형 검색 시스템을 자체 구축함.

**Case 2. 낙관적 업데이트(Optimistic Update)를 통한 UX 개선**
문제 상황 (Problem):

모달 창에서 파트나 담당자를 지정하고 저장할 때, 변경된 데이터를 반영하기 위해 fetchRequests()를 호출하여 화면 전체 데이터를 다시 불러옴.

이로 인해 화면 전체가 깜빡이거나 "데이터 로딩 중..." 화면이 노출되어 사용자 경험(UX)이 저하됨.

**해결 과정 (Action):**

서버에 PUT 요청이 성공하면, 전체 리스트를 다시 조회하는 대신 프론트엔드의 로컬 상태(Local State) 배열에서 해당 요청 건의 데이터만 찾아 수정된 값으로 덮어쓰는 낙관적 업데이트 로직을 적용함.

**결과 (Result):**

불필요한 네트워크 요청(GET)을 제거하여 서버 부하를 줄이고, 모달 창이 닫힘과 동시에 리스트가 깜빡임 없이 0.1초 만에 부드럽게 갱신되는 Seamless한 사용자 경험을 제공함.

--

**Case 3. 미지정 데이터 추적 및 하드코딩 필터 개선**

**문제 상황 (Problem):**

담당자나 파트가 지정되지 않은 데이터가 필터링 조건에 누락되어 목록에 보이지 않는 현상 발생.

수기 입력으로 인한 오타(Human Error) 발생.

**해결 과정 (Action):**

DB에 존재하는 데이터를 기반으로 동적 developerList를 추출하여 select 콤보박스 형태로 입력 방식을 개선.

필터에 명시적으로 NONE(미지정) 옵션을 추가하고, 미지정 건 발생 시 카드 뷰에 AlertTriangle 아이콘과 붉은색 경고 배지를 노출하여 시인성을 높임.

🏛️ 시스템 아키텍처 및 데이터베이스 구조
**1. Data Flow (유사 내역 조회 및 학습)**

Client ➡️ 모달 오픈 요청

Backend ➡️ SIMILAR_NOS 존재 여부 확인

(Yes) ID로 데이터 즉시 로드 ➡️ Client

(No) 텍스트 분석 ➡️ KEYWORD_DICTIONARY 대조 ➡️ 표준 키워드 변환 ➡️ 유사 내역 조회 ➡️ 결과 DB 캐싱 ➡️ Client

Client ➡️ 태그 수기 수정 및 저장

Backend ➡️ KEYWORD_DICTIONARY 업데이트(학습) 및 SIMILAR_NOS 재계산


**2. Database Schema (주요 테이블 확장)**

it_requests (요청 기본 테이블)

KEYWORDS (TEXT): 최종 확정된 해시태그 목록

SIMILAR_NOS (TEXT): 캐싱된 유사 내역 ID 배열

TASK_DETAIL (TEXT): 최종 처리 및 작업 내역 (SQL 쿼리 등 자동 추출 및 수기 작성용)

KEYWORD_DICTIONARY (신규: 키워드 사전 테이블)

RAW_WORD (TEXT, PK): 사용자 원문 (예: "로그인불가")

STD_WORD (TEXT): 시스템 표준어 (예: "계정잠금해제")
